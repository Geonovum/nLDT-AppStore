# Roadmap

## Phase 1
1.	Data dat wordt aangeboden via de App Store, wordt aangeboden via API’s (liefst OGC API *, maar kan ook via oude WMS, WFS als tussentijdse oplossing)
1.	Data verwijzingen (aka Metadata) worden via een Catalogus aangeboden.
1.	Catalogi (aka App Store) in het ecosysteem worden ontsloten via API Records.

## Phase 2
1.	Rekenmodellen worden aangeboden via de App Store, aangeboden via API Processes.
1.	De techniek van het model wordt online beschreven
1.	Rekenmodellen verwijzingen (aka Metadata) worden via een Catalogus aangeboden.
1.	DCAT is gemaakt voor datasets, minder voor Rekenmodules – extra type aanduidingen zijn hiervoor nodig. De vraag ligt bij W3C.
1.	Mogelijk zijn extra velden (extension) nodig in DCAT om rekenmodules te beschrijven
1.	Andere diensten worden ook beschreven in DCAT en aangeboden via de Catalogus.
1.	Ook hiervoor zijn extra type aanduidingen nodig in Metadata.
1.	Mogelijk zijn extra velden (extension) nodig in DCAT om rekenmodules te beschrijven (beoogt publiek, …)

## Phase 3
1.	Catalogi federeren. Het no-wrong-door principe, je kan eender waar zoeken, je vind je data of rekenmodules.
1.	Recepten aanbieden via de Catalogi (App Store)
1.	Ook hiervoor zijn extra type aanduidingen nodig in Metadata.
1.	Alle acties van de App Store worden gelogd volgens logboek dataverwerking principes.
